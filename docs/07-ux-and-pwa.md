# 07. UX と PWA

モバイルファーストで設計し、PWA として提供する。

---

## 7.1. 設計原則

### 7.1.1. 「2 タップで問題開始」

- 通知 / ホーム画面アイコン → ホーム画面 → Drill カードタップ → 即出題
- ログイン画面を挟まない (Clerk の cookie/JWT 自動)
- スプラッシュ画面は最小限

### 7.1.2. 「片手操作」

- ボタンは画面下部に集約 (親指リーチ圏内)
- スクロール不要で 1 問が収まるレイアウト
- 選択肢ボタンは大きめ (48px 以上)

### 7.1.3. 「10 秒でスキップできる任意項目」

- 「なぜそう答えたか」入力は任意、10 秒でスキップ可能
- 自己評価は 1-5 の絵文字ボタン (タップ 1 回)

### 7.1.4. 「読むより解く」

- 解説は折りたたみで表示、ワンタップで次へ進める
- 長文は避ける、図やコードで補う

## 7.5. PWA 対応

### 7.5.1. 必要要素

| 要素 | 内容 |
|---|---|
| `manifest.webmanifest` | アイコン、名前、`display: standalone`、テーマ色 |
| Service Worker | App Shell キャッシュ、オフライン起動 |
| Web Push | daily reminder、streak at risk |

### 7.5.2. Manifest

```json
{
  "name": "Tanren",
  "short_name": "Tanren",
  "description": "エンジニアのための AI 家庭教師",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#0a0a0a",
  "theme_color": "#3b82f6",
  "orientation": "portrait",
  "icons": [
    {
      "src": "/icons/icon-192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/icons/icon-512.png",
      "sizes": "512x512",
      "type": "image/png"
    },
    {
      "src": "/icons/icon-maskable.png",
      "sizes": "512x512",
      "type": "image/png",
      "purpose": "maskable"
    }
  ]
}
```

### 7.5.3. Service Worker の戦略

- **App Shell**: Cache First (オフライン起動のため)
- **API レスポンス**: Network First (新鮮さ優先、フォールバックでキャッシュ)
- **問題の先読み**: Stale While Revalidate (数問先を裏で取得)
- **解答の送信**: Background Sync (電波復帰時に自動送信)

### 7.5.4. オフライン対応の範囲

- **できる**:
  - 以前解いた問題の履歴閲覧
  - 先読みされた問題を解く (解答は復帰時に送信)
  - Insights の Dashboard (直近のキャッシュ)
- **できない**:
  - 新規の問題生成 (LLM 必須)
  - Custom Session の NL パース
  - 詳細な Insights 集計

### 7.5.5. Web Push

- **VAPID キーペア** でサーバー側署名
- 購読情報は DB に保存
- cron で一斉配信
- iOS 16.4+ 対応 (PWA をホーム画面追加時のみ購読可能)

## 7.6. モバイルとデスクトップの UI 分岐

### 7.6.1. モバイル (〜 640px)

- ボタン / 入力は画面下部
- ナビは bottom tab bar
- 問題1つで画面埋まる

### 7.6.2. デスクトップ (1024px+)

- サイドバー ナビ
- 問題 + 解説を 2 カラムで同時表示 (採点後)
- キーボードショートカットがメイン操作

### 7.6.3. タブレット (640-1024px)

- モバイル UI ベースで一部拡張
- ナビは top bar + hamburger

- **デフォルトでダークモード** (エンジニアの好み)
- `prefers-color-scheme` に従って自動切替
- 設定で手動指定可能

## 7.8. アクセシビリティ

- WCAG AA 準拠を目標
- キーボードのみで全機能操作可能
- 色だけでなくアイコン/テキストで情報伝達
- 文字サイズ変更に対応
- `aria-*` 属性を適切に設定

## 7.9. アニメーション

### 7.9.1. 使いどころ

- ページ遷移: 控えめなフェード
- 採点結果: ✅/❌ の軽いバウンス
- Mastery 増加: プログレスバーのスムーズ移行
- 連続日数更新: 軽いセレブレーション

### 7.9.2. 避けるべき

- 過度な装飾アニメーション
- スプラッシュ画面の長いローディング
- 学習の集中を削ぐ演出

### 7.9.3. 実装

- CSS Transitions 中心
- 凝った動きは **Framer Motion** を局所利用

## 7.10. 通知戦略

### 7.10.1. 通知の種類と設計

| 種類 | タイミング | 本文例 |
|---|---|---|
| Daily reminder | 設定時刻 | 「今日の復習 12 問が待ってます」 |
| Streak at risk | 23:00 | 「連続 7 日、途切れる前に 1 問だけでも」 |
| Weekly digest | 日曜 9:00 | 「先週の学びレポートが届きました」 |
| Blind spot unlocked | 随時 | 「Paxos が解禁されました (Quorum を習得済)」 |

### 7.10.2. 通知を嫌われない工夫

- すべての通知をユーザーが個別 on/off 可能
- デフォルトで過度に送らない (Daily + Weekly のみ)
- 本文は具体的 ("12 問" など数字を入れる)

## 7.12. エラー / エッジケース

### 7.12.1. LLM 失敗時

- タイムアウト → キャッシュから代替問題を返す
- 採点失敗 → 再試行 3 回 → ダメなら「後で確認します」扱い (後で自動再採点)

### 7.12.2. ネットワーク切断

- 解答は IndexedDB に保存 → 復帰時に送信
- 「オフライン中です」表示 + リトライボタン

### 7.12.3. 空データ

- 初心者に「今日の復習 0 問」表示 → 代わりに「新しい concept を試そう」誘導
- ドメイン未着手 → 「まずは診断テストから」誘導

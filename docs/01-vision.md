# 01. ビジョン

## 1.1. ひとことで

> **「AI が毎日問題を出し続け、忘れる前に再出題する、エンジニア専用の家庭教師」**

これは個人開発のプロジェクト。作者自身が毎日使って、エンジニアとして鍛えられるための道具。

---

## 1.2. 解きたい課題

エンジニアが継続的に学ぶべき知識は極めて多岐にわたる:

- **プログラミング基礎** — 言語仕様、型、スコープ、メモリ管理、エラー処理
- **データ構造とアルゴリズム** — 配列/ツリー/グラフ/ハッシュ、計算量、定番アルゴリズム
- **OS** — プロセス/スレッド、メモリ管理、スケジューリング、I/O、並行性
- **ネットワーク** — TCP/IP、HTTP、TLS、DNS、WebSocket、REST/gRPC
- **データベース** — RDB/NoSQL、トランザクション、インデックス、クエリ最適化
- **セキュリティ** — 認証/認可、暗号、OWASP Top 10、脅威モデリング
- **分散システム** — CAP、レプリケーション、合意アルゴリズム、スケーリング
- **ソフトウェア設計** — 設計パターン、アーキテクチャ、DDD、テスト戦略、SOLID
- **DevOps/インフラ** — Docker、Kubernetes、CI/CD、クラウド、観測性 (ログ/メトリクス/トレース)
- **実務ツール** — Git 内部、Linux/シェル、エディタ/IDE、デバッガ
- **低レイヤ (応用)** — アセンブリ、キャッシュ、メモリモデル、コンパイラ

すべてを深くやる必要はないが、エンジニアとして成長するには **広く触れて、必要に応じて深掘り**できることが重要。

既存の学習手段にはそれぞれ限界がある:

| 手段 | 限界 |
|---|---|
| 書籍・記事を読むだけ | 受動的で身につかない。数日で忘れる |
| Anki などの静的カード | 同じ問題が同じ形で出て飽きる。作成コストも高い |
| ChatGPT / Claude との壁打ち | その場では理解するが、学習履歴が残らず定着しない |
| 資格試験 | ゴールが明確だが範囲が固定され、日々の鍛錬に使いにくい |
| オンライン講座 | 座学中心、能動的な解答機会が少ない |

---

## 1.3. Tanren のコアバリュー

1. **能動的解答が中心** — 読むだけ学習を捨て、「問題を解く」に全振り
2. **AI による動的生成** — 同じ概念を違う角度で無限に問える
3. **科学的な記憶定着** — 忘却曲線に合わせて再出題 (Spaced Repetition, FSRS)
4. **自分の学習状態が見える** — 弱点・盲点・忘却中の概念を診断できる
5. **ユーザー指定の出題** — 「こういう問題を出して」と自然言語で指定可能
6. **どこでも続けられる** — PWA でモバイル/デスクトップ両対応、通勤中にも1問

---

## 1.4. 対象ユーザー

### メイン: 作者本人 (1年目エンジニア想定)

この Tanren は作者 (1年目エンジニア) 自身が使うために作る。
想定する学習者像:

- エンジニアとして働き始めて間もない〜数年目
- CS の基礎 (OS/ネットワーク/DB/セキュリティなど) を体系的に鍛え直したい
- 毎日少しずつ、継続的に学習したい
- 自分の弱点・盲点を可視化したい

### 想定するレベル感

Tanren では問題の難易度を段階的にカバーする:

| レベル | 対象者像 |
|---|---|
| **beginner** | プログラミングは書けるが CS 基礎はこれから学ぶ人 |
| **junior** | 1-2 年目の現役エンジニア、基礎を固めたい |
| **mid** | 3-5 年目、応用と深掘りをしたい |
| **senior** | 6 年目〜、専門分野以外の鈍りを防ぎたい |

**MVP では beginner と junior レベルを中心にカバー**。自分が最初から使えることを最優先。

### 将来の拡がり

もし将来公開する気になったら、このプロダクトで恩恵を受けそうな人:

- 学生・ブートキャンプ生 — 就職に向けた基礎固め
- 現場エンジニア全般 — 範囲を問わず日々の鍛錬
- ブランクから復帰するエンジニア — 体系的に取り戻したい

ただし公開は未定で、まずは個人用途として完結させる。

## 1.5. 既存ツールとの違い

| 既存ツール | Tanren との違い |
|---|---|
| Anki / Quizlet | 問題が AI で動的生成、エンジニア特化タクソノミ、採点が LLM で柔軟 |
| Exercism / LeetCode | コーディング特化ではなく**知識全般**、短時間で完結 |
| ChatGPT / Claude | **学習履歴が残り、科学的に再出題**される |
| Duolingo | 語学ではなくエンジニア専用、問題の深さと柔軟性が違う |
| 書籍 | インタラクティブ、能動的、個人最適化 |

---

## 1.6. 価値提案

> **「毎日 15 分、Tanren が出す問題に答えるだけで、エンジニアとしての基礎〜応用知識が科学的に鍛えられる。弱点は AI が見つけ、覚えた頃を見計らって再出題してくれる。」**

## 1.7. コア体験 (Day in the Life)

```
朝の通勤 (8:00)
  ├─ スマホで Tanren を開く
  ├─ 「今日の復習 12 問」をタップ
  └─ 10 分で終了。TCP 輻輳制御の問題で部分正解 → 解説を読む

午後の休憩 (14:00)
  └─ プッシュ通知: 「OS メモリ管理の新しい角度の問題が届きました」
     タップして 3 問解く

夜 (22:00)
  ├─ 「最近 TLS 1.3 の業務が多いので深掘りしたい」
  ├─ カスタムセッション: 「TLS 1.3 の trade_off を junior レベルで 5 問」
  └─ 15 分でセッション完了。Insights で TLS サブドメインの mastery が 41% → 52% に

週末 (土曜)
  └─ Insights の Weekly Digest を確認
     「今週は OS で成長、Network は未着手」を見て Deep Dive に誘導される
```

---

## 1.8. 成功の定義 (個人開発としての)

このプロジェクトが成功したと言える条件:

1. **自分が毎日使っている** — 継続して問題を解いている
2. **学習が進んでいる実感がある** — 以前分からなかった概念が分かるようになる
3. **Insights で成長が見える** — mastery 数が増えていく
4. **作ること自体が楽しい** — 作る過程で Next.js / AI / SRS の知見が深まる

ユーザー数やマネタイズは目標ではない。**自分が賢くなるための道具として成立すれば十分**。

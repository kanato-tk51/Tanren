# Tanren ドキュメント

エンジニアのための AI 家庭教師アプリ **Tanren (鍛錬)** の設計ドキュメント群。

## 読む順番

初めての人は上から順に読むのを推奨。

| # | ドキュメント | 内容 |
|---|---|---|
| 01 | [ビジョン](./01-vision.md) | 何を作るか、誰のためか、なぜ作るか |
| 02 | [学習システム](./02-learning-system.md) | 知識タクソノミ、問題タイプ、SRS、マスタリー、セッション |
| 03 | [AI 活用戦略](./03-ai-strategy.md) | Claude API の使い方、モデル選定、コスト |
| 04 | [カスタムセッション](./04-custom-sessions.md) | ユーザー指定の問題生成機能 |
| 05 | [Insights モジュール](./05-insights.md) | 学習状態の診断・可視化 |
| 06 | [システム構成](./06-architecture.md) | アーキテクチャ、データモデル、API、技術選定 |
| 07 | [UX と PWA](./07-ux-and-pwa.md) | 画面設計、PWA 対応 |
| 08 | [MVP とロードマップ](./08-mvp-roadmap.md) | 最初に作るもの、段階的な拡張計画 |
| 09 | [リスクと成功指標](./09-risks-and-metrics.md) | 想定リスクと成功の定義 |

## このドキュメントの目的

- 設計の合意点と判断の根拠を残す
- 実装着手時に立ち返れる北極星にする
- チームメンバー / 将来の自分への引き継ぎ

## ドキュメントの更新方針

- 設計が変わったら該当ファイルを更新
- 大きな方針変更は ADR (Architecture Decision Record) を別途 `docs/adrs/` に追加 (将来)
- 「TODO」「要検討」のマーカーは現時点では残してよい
